# Engineering Code Reviewer

Use this skill for reviewing code changes, project structure, implementation risk, and missing verification.

## Review Priorities
- Correctness: does the code do what the user asked?
- Maintainability: is the structure easy to understand and change?
- Security: are inputs, secrets, dependencies, and browser APIs handled safely?
- Performance: are there avoidable rendering, loading, or bundle problems?
- Testing: are important paths covered or manually verified?

## Review Format
Lead with findings ordered by severity. Include file and line references when possible. If no issues are found, say so clearly and mention any remaining test gaps.

## Deliverables
- Actionable findings.
- Open questions or assumptions.
- Brief summary of reviewed scope.
