# Project Instructions for Codex

## Language
- Always reply to the user in Simplified Chinese.

## Current Stage
- This repository is in the planning/setup stage.
- Do not assume a website framework exists until `package.json`, `src/`, or an equivalent app entry is present.
- Before creating website code, confirm the intended site type, audience, content scope, and preferred stack when those choices are unclear.

## Project Goals
- Build a clean, maintainable, production-ready website.
- Prefer simple architecture and clear file organization.
- Keep future work easy to inspect, run, test, and hand off.

## Skills To Use
- Use `engineering-frontend-developer` for frontend implementation, responsive layout, accessibility, and performance work.
- Use `design-ui-designer` for visual direction, layout systems, UI consistency, and design decisions.
- Use `engineering-code-reviewer` for code review, risk checks, maintainability, and test coverage.
- Use `product-manager` for scope definition, user goals, feature prioritization, and roadmap planning.
- Use `marketing-seo-specialist` for SEO structure, metadata, content hierarchy, and organic growth planning.

## Development Style
- Briefly explain the plan before editing files.
- Prefer small, focused changes.
- Do not add new dependencies unless they clearly reduce complexity or match the chosen framework.
- Follow existing project conventions once a framework is added.
- After edits, summarize changed files and how to run or preview the result.

## Website Standards
- Mobile-first responsive design.
- Semantic HTML and accessible interactions.
- Good SEO basics: titles, descriptions, heading structure, crawlable content, and useful metadata.
- Fast loading and optimized assets.
- Clean UI suitable for production.

## Commands
- Prefer `npm` if `package-lock.json` exists.
- Prefer `pnpm` if `pnpm-lock.yaml` exists.
- Prefer `yarn` if `yarn.lock` exists.
- Run available checks such as build, lint, or tests before final delivery when a project exists.
