# Site Direction

## Chosen First Direction
Build a niche directory and comparison website for AI and software tools used by cross-border e-commerce sellers, Shopify store owners, Amazon sellers, and independent online merchants.

Working concept:

> A searchable, SEO-friendly tool database that helps online sellers find, compare, and choose tools for product research, listing optimization, ad creatives, SEO, customer support, email marketing, analytics, and automation.

## Why This Direction

This direction fits the project goals:

- Revenue potential: many SaaS tools have affiliate, partner, or sponsorship programs.
- SEO potential: users search for specific tool categories, comparisons, alternatives, and "best tools for..." queries.
- Automation potential: tool data can be collected from official websites, RSS feeds, changelogs, APIs, pricing pages, public directories, and manual submissions.
- Maintainability: content is structured as records instead of only articles.
- Long-term expansion: can grow from directory pages into comparison pages, guides, rankings, newsletters, and lead generation.

## Target Audience

Primary audience:

- Cross-border e-commerce sellers.
- Shopify store owners.
- Amazon, Etsy, eBay, or TikTok Shop sellers.
- Small teams running independent online stores.

Secondary audience:

- E-commerce agencies.
- Freelancers building stores for clients.
- SaaS buyers researching alternatives.

## Core User Problems

Users need to:

- Find reliable tools for a specific e-commerce task.
- Compare pricing, features, pros, cons, and use cases.
- Discover alternatives to tools they already know.
- Avoid wasting time testing low-quality tools.
- Understand which tool fits their store size, budget, and workflow.

## First Monetization Paths

Prioritize these in order:

1. Affiliate links to SaaS tools.
2. Sponsored placements, clearly labeled.
3. Paid tool submissions or featured listings.
4. Newsletter sponsorships after traffic grows.
5. Lead generation for agencies or service providers.

Avoid at the beginning:

- Paid memberships.
- Complex user accounts.
- Selling courses before traffic exists.
- Too many ad placements that reduce trust.

## Content Acquisition Strategy

Use allowed and maintainable sources:

- Official tool websites.
- Pricing pages.
- Changelogs and release notes.
- RSS feeds.
- Public APIs when available.
- Product Hunt or similar public listings, subject to terms.
- Manual submissions from tool vendors.

Do not:

- Copy full articles from other websites.
- Republish scraped descriptions without transformation.
- Auto-publish low-quality generated pages.
- Index pages with too little unique information.

## Unique Value To Add

Every public indexed page should add at least one layer of original value:

- normalized pricing and feature fields,
- category-specific scoring,
- pros and cons,
- best-fit user type,
- alternatives,
- comparison tables,
- update history,
- source attribution,
- editorial notes,
- filtering and sorting.

## Recommended Language Strategy

Start with English-first pages if the goal is affiliate revenue from global SaaS tools.

Possible later expansion:

- Chinese summaries for Chinese-speaking sellers.
- bilingual interface.
- separate localized SEO pages only after the English structure works.

## Initial Niche Scope

Start with these categories:

- Product research tools.
- Listing and description generators.
- SEO and keyword tools.
- Ad creative and image/video generation tools.
- Email marketing and retention tools.
- Customer support chatbots.
- Analytics and attribution tools.
- Workflow automation tools.

Do not cover every AI tool category at launch. Stay focused on e-commerce use cases.

## Positioning

Not a generic AI tools directory.

Position as:

> AI and growth tools for e-commerce sellers.

This is narrower, easier to explain, and better for SEO than a broad "AI tools directory".

## Success Criteria For MVP

The first version is successful if:

- at least 50 tool records exist,
- at least 8 category pages are generated,
- at least 20 high-value detail pages are indexable,
- at least 5 comparison/alternative pages exist,
- every indexed page has unique useful information,
- the admin can add/edit tools without touching code,
- import/update logs show where data came from,
- SEO metadata and sitemap are generated automatically.

## Open Questions

- Should the site brand be English, Chinese, or bilingual?
- Should we target Shopify first, Amazon first, or all e-commerce sellers?
- Do you prefer affiliate revenue first or paid submissions first?
- Should automatic updates require manual review before publishing?
